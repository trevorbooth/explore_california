Git Essentials

  • Chapter 1

    Version Control Systems have come along way from where they started. Using git will be a large part of sharing information and code, all easily and free.

  • Chapter 2

    This Chapter was all about installing Git to your computer. Configuring it, to help with usability and make it your own. Also, some helpful tips if you're are stuck and need help; you can type "git help" in the terminal/command prompt giving you a list of help.

  • Chapter 3

    Chapter 3, we started by doing our first Initialize and understanding how to start a repository. Commit log is a place for notes showing what you've changed in your work. There is a right and wrong standard for leaving commit notes.

  • Chapter 4

    Understanding the 3 stages of the Git tree architect and how the flow of work happens. Starting at the working stage, moving to the index, and then commiting your work into the repo. Git uses a checksum algorithm to see what changes have been applied to the workflow.The HEAD is the most recent point of work commited in the repo.

  • Chapter 5

    Chapter 5, was all about making changes to files. The basics of moving files, deleting, renaming, and seeing what was changed in the files. They're many ways to do most of these operations aswell.

    How to create folders that will be ignored. When to ignore certain files.

  • Chapter 9

    How to move about the commit tree and blobs. Finding old commits, comparing commits, and also finding keywords in the commit log. Looking at the log in an easy way or finding a commit by time.

  • Chapter 10

    Basics of branches, when to use them, why we use them, and the benefits. How to create a branch and how to move back and fourth between master and the new branch. Naming branches, comparing, and deleting them. Also how to see what branch you are on all the time.

  • Chapter 11

    The fast-forward merge is fairly easy. True merge is a little more legnthy still simple. Conflicts can often resolve themselves, if not you must go in and manually do it or use a merge tool.

  • Chapter 12

    Stashing is a useful tool you want to switch back and fourth between branches. Instead of having to commit the work you're unsure of you can stash it. You can pop it out or take it out and leave a copy in the stash.
